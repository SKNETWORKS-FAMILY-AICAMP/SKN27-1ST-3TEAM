## 2. 프로젝트 개요

### 2-1. 프로젝트명

대한민국 전기차 전환 분석 통합 대시보드

---

### 2-2. 프로젝트 소개

본 프로젝트는 자동차 탄소배출 문제와 전기차 전환 정책의 관계를 데이터로 분석하기 위해 기획하였다.

기온, 수송 부문 CO₂ 배출량, 자동차 등록 현황, 전기차 충전 인프라, 보조금 정책 데이터를 하나의 구조 안에서 통합하여 대한민국 전기차 전환 흐름을 종합적으로 분석하는 대시보드 플랫폼을 구축한다.

단순 통계 조회가 아니라, 여러 지표를 연결해 정책 효과와 지역 격차를 비교·분석하는 것을 목표로 한다.

---

### 2-3. 프로젝트 배경

<img src="https://contents.kyobobook.co.kr/sih/fit-in/458x0/pdt/9788984074491.jpg" width="300px" align="right">

『6도의 멸종』은 기온 상승이 환경에 미치는 심각한 영향을 경고한다.
본 팀은 이를 자동차 부문의 탄소배출 문제와 연결해 보았다.

현재 전기차 전환과 관련된 데이터는 여러 기관에 분산되어 있다.이로 인해 다음과 같은 한계가 존재한다.

- 전기차 증가가 실제로 탄소배출 감소로 이어지는지 분석하기 어려움
- 보조금이 많은 지역이 등록 대수도 높은지 비교하기 어려움
- 지역 간 충전 인프라 격차를 한 번에 파악하기 어려움

이러한 문제를 해결하기 위해 공공 데이터를 통합 분석하는 시스템을 기획하였다.

---

### 2-4. 프로젝트 목표

1. **전기차 보급과 탄소배출의 관계 분석**→ 전기차 등록 추이와 수송 CO₂ 배출량을 함께 시각화하여 변화 흐름을 비교한다.
2. **지역별 인프라 격차의 정량적 분석**→ 전기차 대비 충전기 수 등 계산 지표를 통해 지역 간 불균형을 도출한다.
3. **보조금 정책 효과 검증**→ 지역별 보조금 수준과 전기차 등록 대수를 비교 분석한다.
4. **사용자 관점 정보 제공**
   → 전기차 구매를 고려하는 소비자가 인프라 및 보조금 정보를 객관적으로 비교할 수 있도록 지원한다.

또한 분석 대상으로 현대 기업을 선정하였다.
국내 시장 점유율이 높고 전기차 기술을 선도하며 산업 구조와 밀접하게 연결되어 있기 때문이다.

---

### 2-5. 기대 효과

1. 전기차 보급과 탄소배출 간의 관계를 데이터 기반으로 이해할 수 있다.
2. 지역별 보조금 및 충전 인프라 수준을 비교함으로써 합리적인 의사결정을 지원한다.
3. 정책 효과를 수치로 검증하여 전기차 전환 흐름에 대한 인사이트를 도출한다.
4. 단순 조회 사이트가 아닌, 대한민국 전기차 전환 구조를 분석하는 통합 플랫폼으로 활용할 수 있다.

# 8. 한계점 및 향후 과제

한계점 및 향후 과제

1. 실시간 연동의 부재 → 실시간 데이터 처리 구조 개선 필요
   현재 시스템은 정적 데이터 기반으로 동작하여 데이터가 즉각적으로 반영되지 않는 한계가 있습니다. 향후에는 API 연동 또는 주기적 크롤링 및 자동 갱신 로직을 적용하여 데이터가 실시간으로 업데이트될 수 있도록 개선할 계획입니다. 이를 통해 서비스의 신뢰성과 활용도를 높이고자 합니다.
2. UI 구성의 단순 나열 구조 → 사용자 중심 인터페이스 개선 필요
   그래프가 화면 하단으로 단순 나열되어 가독성이 떨어지는 문제가 있습니다. 향후에는 상단 메뉴바 또는 사이드바를 활용하여 사용자가 원하는 그래프만 선택적으로 확인할 수 있도록 구조를 개선할 예정입니다. 이는 사용자 경험을 향상시키는 중요한 개선 요소입니다.
3. 목적 설계의 구체성 부족 → 명확한 타겟 및 목표 설정 필요
   프로젝트의 목표와 타겟이 다소 포괄적으로 설정되어 있어 기능 설계의 방향성이 분산되는 경향이 있습니다. 향후에는 구체적인 사용자 타겟과 핵심 목표를 명확히 정의하여, 그에 맞는 기능 중심으로 발전시킬 계획입니다. 이는 서비스 완성도를 높이기 위한 중요한 과제입니다.

# 9. 한 줄 회고

**27기_이혜림**
이번 프로젝트를 통해 순서와 절차의 중요성, 그리고 그것을 미리 계획하는 일이 더욱 중요하다는 것을 깨달았습니다. 미래를 알 수는 없지만, 대비하고 방향을 설정해두면 이후 과정에서 발생하는 문제도 무리 없이 해결할 수 있다는 점을 배웠습니다. 또한 협업에서는 내가 무엇을 할지보다, 내가 어떤 방향으로 가고 있는지를 아는 것이 더 중요하다는 것을 다시 한 번 느끼게 되었습니다.

**27기_김주영**
끝나고 나서야 전체 흐름이 보이기 시작한 것 같습니다.
2차 단위 프로젝트에서는 의사소통을 더욱 적극적으로 진행하여 전체 흐름에 맞춰 보다 더 정확한 역할 수행을 하겠습니다.

**27기_박준희**
첫 프로젝트를 경험하면서 전체적인 진행 방향과 흐름을 이해할 수 있었습니다.
배운 내용을 다음 프로젝트에서 활용할 수 있도록 꾸준히 학습하는 게 중요할 것 같습니다.

**27기_김필주**
비전공자로서 새롭게 보는 구현이나 기능에 굉장히 공부가 많이 되었고 이번에 화면 구현 중 데이터를 sql에 직접 연결하는 기회가 있었는데 어렴움을 겪어 결국 더미 데이터로 작성을 진행했기에 이번 기회에 공부하여 sql을 더 자유롭게 쓰고싶은 마음가짐입니다.

**27기_김재묵**
개발 과정에서 단순히 코드를 짜는 것만큼이나 중요한 것은 왜 이 프로젝트를 했는가 에 대한 기획 의도를 명확히 파악하는 것이었습니다. DB 연결 등 초기 단계에서 마주한 에러들은 저를 당황하게 하기보다 장기적으로 저에게 더 도움이 될 것이라고 생각했고 특히 협업 시에는 애매하게 아는것 보다 모르는 것을 확실하게 이해하는 것이 더 중요하다는 것을 느꼈습니다. 서로의 의도를 명확히 인지하고 작업할 때 비로소 팀워크가 완성된다는 소중한 교훈을 얻었습니다.
